# NovelForge 重构追踪（对齐 `docs/11.md`）

更新时间：2026-05-02  
评估方式：基于当前仓库代码与迁移脚本的静态核查（不推测未实现行为）

## 1. 评估口径

- `已完成`：核心机制已落地，链路可追踪，剩余仅为边角优化。
- `部分完成`：已有主干实现，但关键能力缺口仍影响 `docs/11.md` 目标。
- `未开始`：仅有概念或 UI 痕迹，缺少后端/数据/闭环实现。

## 3. 新旧逻辑冲突清单（更新后）

1. （已消解）路由模型层级冲突：已落地 `task -> model pool -> provider/model` 兼容链路。  
2. （已消解）连续性上下文深度冲突：章节关键任务已强制 `deep`，并补齐 `context_incomplete` 可解释告警。  
3. （已消解）技能回写粒度冲突（阶段1范围）：已覆盖情绪/场景环境/关系温度三类结构化状态。  
4. （已消解-阶段2）场景编排闭环冲突：已接入 `post_tasks` 与后置执行器，生成后可自动 review/extract 并写审计。  
5. （已消解-阶段3）状态 taxonomy 扩展冲突：动作/着装/公开信息边界已进入统一结构化分类。
