## 关键重构点
1. 冻结区冲突策略重构（旧逻辑 -> 新逻辑）
- 旧逻辑：检测到冻结区冲突时，仅将 `persistMode` 从 `formal` 降级到 `derived_review`。
- 新逻辑：检测到冻结区冲突后立即阻断 pipeline，返回 `PIPELINE_FREEZE_CONFLICT`。
- 原因：避免“继续生成但仅降级持久化”的双语义冲突，统一为“冲突即阻断”。

2. Blueprint 分区校验器
- 新增跨分区重叠校验：同一条目不能同时属于冻结区/承诺区/探索区。
- 后端错误码：`BLUEPRINT_CERTAINTY_ZONES_OVERLAP`。
- 前端联动：step-08 出现冲突时阻断保存/标记完成/晋升，并展示冲突明细。

3. UI 冻结区冲突提示联动
- Editor 页新增 `PIPELINE_FREEZE_CONFLICT` 友好建议映射：
  - 引导到“蓝图 > 章节路线 > 确定性分区”修复冻结区冲突。
- Blueprint 页在编排阶段错误与保存错误中统一展示阻断提示。

## 兼容策略

- 新流程以显式 DTO 为主，且在保存阶段执行校验，禁止冲突状态继续沉淀。
