# 重构轨迹（11.md 对齐）

## [R-001] Review Trail 聚合化（chapter 维度）
- **日期**：2026-05-02
- **目标**：将 `Review Trail` 从“仅人工审查动作”升级为“五对象模型”所需的可追溯时间线基础（对齐 `Review Trail` 方向）。
- **范围**：
  - `src-tauri/src/services/review_trail_service.rs`
  - `docs/22.md`（本记录）
- **冲突描述**：
  - 旧逻辑：`get_review_trail` 仅查询 `user_review_actions`，无法体现 AI 生成链路与反馈闭环状态。
  - 新逻辑需求：chapter 轨迹应覆盖 AI 生成记录、自动推进结果、用户审查动作与反馈闭环结果，支持“为什么现在是这个结果”的追溯。
- **决策**：采用“新逻辑优先 + 渐进式迁移（兼容层）”
  - 保持既有 command 入参与返回 DTO 不变（兼容前端调用）。
  - 在 `entity_type = chapter` 时，聚合 `user_review_actions + ai_pipeline_runs + feedback_events`。
  - 非 chapter 查询保持旧行为，仅返回 `user_review_actions`。
- **实现**：
  - 重构 `ReviewTrailService.get_review_trail` 为聚合查询并统一时间倒序输出。
  - 新增聚合来源映射：
    - `ai_pipeline_runs` → `entity_type=ai_pipeline_run`，`action=ai_*`
    - `feedback_events` → `entity_type=feedback_event`，`action=feedback_*`
  - 保留旧结构：`ReviewTrailRecord` 字段不变，通过 `detail` 承载来源细节。
  - 新增单元测试 2 条：
    - chapter 维度聚合包含三类来源记录。
    - 非 chapter 维度保持旧逻辑（仅用户审查动作）。
- **验证**：
  - `cargo test --manifest-path src-tauri/Cargo.toml review_trail_service::tests` 通过（2 passed, 0 failed）。
- **风险**：
  - 当前 `feedback_events` 仅保存最终状态，历史状态迁移（open→ack→resolved）的完整事件序列仍不完整。
  - 前端“审查轨迹”标签文案目前主要针对 `approved/rejected/edited`，新增 `ai_* / feedback_*` 动作显示将走兜底文案。
- **下一步**：
  - 在不改接口的前提下，补充前端 `review action` 展示映射，提升可读性。
  - 评估是否为 `feedback_events` 增加状态迁移历史表，以支撑完整闭环复盘。

## [R-002] 技能编排与状态账本冲突修复（S1/S2/S3 批次）
- **日期**：2026-05-02
- **目标**：修复 `docs/11.md` 定义的 6 类核心冲突，落地“五对象模型 + 技能编排系统”的关键运行语义（workflow 编排、state ledger 回写责任、context completeness 硬门禁、post-task 技能化、策略字段运行时消费）。
- **范围**：
  - `src-tauri/src/services/skill_registry.rs`
  - `src-tauri/src/services/ai_pipeline/orchestrator.rs`
  - `src-tauri/src/services/ai_pipeline/task_handlers.rs`
  - `src-tauri/src/services/ai_pipeline/runtime_state_writer.rs`
  - `src-tauri/src/services/ai_pipeline/prompt_resolver.rs`
  - `src-tauri/src/services/ai_pipeline/post_task_executor.rs`
  - `src-tauri/src/services/ai_pipeline/continuity_pack.rs`
  - `src-tauri/src/services/project_service.rs`
  - `src-tauri/src/services/ai_service.rs`
  - `src-tauri/src/commands/skill_commands.rs`
  - `src/types/ai.ts`
  - `src/components/settings/AiStrategyPanel.tsx`
  - `docs/architecture/windows-desktop-architecture.md`
  - `docs/11.md`
  - `docs/22.md`（本记录）
- **冲突描述**：
  - workflow 技能被当作普通 skill-id 平铺激活，缺少阶段语义。
  - `stateWrites` 多技能冲突时无责任归属，回写来源不可追溯。
  - `requiredContexts` 与 `availableContexts` 构建逻辑分散，存在误过滤与漏选。
  - post-task 仅靠路由静态配置，无法跟随技能包动态编排。
  - 上下文缺层仅 warning，不区分关键链路阻断策略。
  - `windowPlanningHorizon` 在运行时无明确消费点。
- **决策**：采用“新逻辑优先 + 渐进迁移”
  - 新增字段全部向后兼容（`Vec` 默认空 / `bool` 默认值），不破坏旧技能与旧项目数据。
  - 关键路径按优先级落地：S1-2 → S1-3 → S1-1 → S2-5 → S2-4 → S3-6。
  - 保留路由 `post_tasks` 作为回退层；技能声明优先。
- **实现**：
  - S1-2：
    - `SkillManifest`/`SelectedSkills` 增加 `StateWriteDeclaration` 聚合能力。
    - `task_handlers::record_runtime_story_states` 按 `workflow > extractor > capability > policy > review` 去重选主写入。
    - `build_runtime_story_state_input` 增加 `source_skill_id`，并写入 payload 字段 `sourceSkillId`。
  - S1-3：
    - `ContinuityPack` 增加 `get_available_context_keys()`。
    - `orchestrator` 先编译 preselection continuity pack，再推断 `availableContexts`。
    - 请求级 `skillSelection.availableContexts` 覆盖会发事件 `PIPELINE_AVAILABLE_CONTEXTS_OVERRIDDEN`。
    - `select_skills_for_task_with_context` 记录 `filtered_skills`（含 `reason/missing_contexts`），route 元数据回传 `filteredSkills`；关键 skill 缺上下文发 `PIPELINE_SKILL_CONTEXT_FILTERED`。
  - S1-1：
    - `SkillManifest` 新增 `workflow_stages`。
    - `SelectedSkills` 新增 `get_workflow_orchestration()`。
    - `PromptResolver` 区分 workflow/capability 渲染：新增 `Workflow Skill Context`，并在 orchestration 区输出阶段链。
    - `resolve_generation_workflow_stack` 不再把非 chapter 任务强行注入 workflow stack。
  - S2-5：
    - `AiStrategyProfile` 新增 `enforce_context_completeness`（默认 `true`）。
    - 关键任务（`chapter.draft/chapter.continue`）缺层硬阻断，错误码 `PIPELINE_CONTEXT_INCOMPLETE_BLOCKED`。
  - S2-4：
    - `SkillManifest` 新增 `post_tasks`。
    - `SelectedSkills` 新增 `get_aggregated_post_tasks()`。
    - `orchestrator` 优先技能 postTasks，缺省回退路由 post_tasks，并记录 `postTaskSources`。
    - `PostTaskExecutor` 结果增加 `source_skill_id` 追溯字段。
  - S3-6：
    - `window_planning_horizon` 接入 ContinuityPack 编译（窗口层增加“窗口规划地平线: N 章”上下文行）。
    - 前端策略面板新增“上下文完整性硬门禁”配置项，策略 DTO 同步字段。
- **验证**：
  - Rust 单测通过：
    - `cargo test --manifest-path src-tauri/Cargo.toml skill_registry::tests`
    - `cargo test --manifest-path src-tauri/Cargo.toml runtime_state_writer::tests`
    - `cargo test --manifest-path src-tauri/Cargo.toml post_task_executor::tests`
    - `cargo test --manifest-path src-tauri/Cargo.toml continuity_pack::tests`
    - `cargo test --manifest-path src-tauri/Cargo.toml prompt_resolver::tests`
    - `cargo test --manifest-path src-tauri/Cargo.toml task_handlers::tests::persist_task_output_with_runtime_state`
    - `cargo test --manifest-path src-tauri/Cargo.toml orchestrator::tests`
    - `cargo test --manifest-path src-tauri/Cargo.toml project_service::tests::ai_strategy_profile_save_and_get_roundtrip_succeeds`
  - 前端类型检查通过：
    - `npm run typecheck`
  - 集成契约测试：
    - `node --import tsx --test --test-concurrency=1 tests/integration/context-completeness.test.ts tests/integration/continuity-pack-contracts.test.ts tests/integration/scene_aware_orchestration.test.ts`
    - 受环境限制失败：`spawn EPERM`；尝试申请沙箱外执行被平台额度限制拒绝（非代码逻辑失败）。
- **风险**：
  - 当前 integration 合同测试未在本轮通过执行，仍有“字符串契约与实际运行事件细节不一致”的残余风险。
  - `workflowStages/postTasks` 对现有技能是可选字段，历史技能未补齐时会走默认/回退语义。
  - `windowPlanningHorizon` 本轮已接入 continuity 编译文案层；若需更强语义（例如驱动 recent continuity 查询窗口），需下一轮补强。
- **下一步**：
  - 补一组端到端 pipeline 集成测试，覆盖 `filteredSkills/postTaskSources/sourceSkillId/context gate` 运行态断言。
  - 为内置 workflow/extractor 技能补充 `workflowStages/postTasks` 声明，减少默认推断路径。
