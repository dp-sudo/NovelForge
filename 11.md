任务流程
Step 1: 类型先行
行动: 在 src/types/ai.ts 中声明/确认 AiStrategyProfile 接口。

约束:

字段命名与 spec 保持一致（defaultWorkflowStack, reviewStrictness, defaultCapabilityPack, maxConcurrentDrafts, autoPersistPolicy）
每个字段附带 TypeScript 注释说明用途
如无现有类型，立即创建；如有冲突，以 spec 为准
验收: TypeScript 编译器通过，无类型错误。

Step 2: API 扩展
行动: 在 src/api/settingsApi.ts 中添加项目级 AI 策略的读写接口。

约束:

函数命名：saveProjectAiStrategy / getProjectAiStrategy
入参/出参类型与 AiStrategyProfile 对齐
与现有 saveProjectSettings / getProjectSettings 风格保持一致
底层写入到 project.sqlite 的 projects 表 ai_strategy_profile 字段
验收: API 函数导出，可被 SettingsPage 调用。

Step 3: 组件构建
行动: 创建 src/components/settings/AiStrategyPanel.tsx。

结构要求:

AiStrategyPanel
├── 面板标题区（"AI 策略配置"）
├── 默认工作流栈配置区
├── 审查严格度配置区
├── 默认能力包配置区
├── 最大并发草稿数配置区
├── 自动持久化策略配置区
└── 保存/重置按钮区
UI 约束:

遵循 SettingsPage 现有 UI 风格（卡片布局、分组标题、说明文字）
使用现有 SliderControl / SelectControl / ToggleControl 组件
每个配置项必须有说明文字，解释该项对 AI 生产的影响
审查严格度：Slider 0-4，分别对应 relaxed / standard / strict / pedantic / exhaustive
自动持久化策略：Select 三选一（auto / supervised / confirm）
验收: 面板可渲染，配置项可交互，数据可保存/读取。

Step 4: 入口接入
行动: 在 SettingsPage.tsx 的 tab 列表中新增 aiStrategy。

约束:

Tab 顺序：建议放在「模型」「路由」「技能」之后，「编辑器」之前
Tab 图标：使用 IconSettings 或类似图标，不另引入新图标包
Tab 标签："AI 策略"
激活时渲染 <AiStrategyPanel projectId={currentProject.id} />
验收: 设置页面显示 8 个 tabs，点击「AI 策略」切换到对应面板。

技术约束
不发明新模式: 复用 SettingsPage 现有状态管理（useState + useEffect 加载模式）
不拆分过细: AiStrategyPanel 保持单文件组件，内部按 section 分割
向后兼容: 项目无 AI 策略配置时，UI 显示默认值而非报错
验收标准
Settings 页面 tab 栏显示「AI 策略」入口
切换到 AI 策略面板显示 5 个配置区
修改配置后保存，数据写入 project.sqlite 的 ai_strategy_profile JSON 字段
重新打开设置页面，已保存配置正确回填
