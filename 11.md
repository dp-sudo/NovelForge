# Task 1: 建立项目级 AI 策略权威源

> **冲突声明**: 如有冲突，以 `docs/superpowers/specs/2026-04-30-ai-production-system-design.md` 定义为准。

---

## 1. 任务目标

建立项目级 AI 策略的运行期权威源，以 `project.sqlite` 为唯一真相源，确保 AI 生产行为可被项目级配置控制。

### 1.1 当前问题

| 现状 | 问题 |
|------|------|
| 项目级 AI 配置缺失 | 无法按项目定制 AI 行为 |
| 无权威存储位置 | 配置散落各处，无法统一管理 |
| 与应用级配置混淆 | 项目设置与应用设置边界不清 |

### 1.2 目标状态

| 目标 | 说明 |
|------|------|
| 权威源确立 | `project.sqlite` 为项目级 AI 策略的运行期真相源 |
| 类型完整 | `AiStrategyProfile` 包含所有项目级 AI 配置字段 |
| 读写闭环 | 后端 Service + Tauri Command + 前端 API 完整链路 |
| 冷启动分离 | `project.json` 仅用于冷启动，不参与运行期 |

---

## 2. 类型体系

### 2.1 AiStrategyProfile（项目级 AI 策略）

| 字段 | 类型 | 说明 |
|------|------|------|
| `automationDefault` | `string` | 默认自动化档位（`auto` / `supervised` / `confirm`） |
| `reviewStrictness` | `number` | 审查严格度（1-5） |
| `defaultWorkflowStack` | `string[]` | 默认工作流栈（技能包 ID 列表） |
| `alwaysOnPolicySkills` | `string[]` | 常驻策略技能（术语锁定、禁用词等） |
| `defaultCapabilityBundles` | `string[]` | 默认能力包（人物表现、场景环境等） |
| `stateWritePolicy` | `string` | 状态写入策略（`chapter_confirmed` / `manual_only`） |
| `continuityPackDepth` | `string` | 上下文编译深度（`minimal` / `standard` / `deep`） |
| `chapterGenerationMode` | `string` | 章节生成模式（`draft_only` / `plan_draft` / `plan_scene_draft`） |
| `windowPlanningHorizon` | `number` | 窗口规划视野（未来章数） |

### 2.2 默认值

| 字段 | 默认值 |
|------|--------|
| `automationDefault` | `"supervised"` |
| `reviewStrictness` | `4` |
| `defaultWorkflowStack` | `["chapter.plan", "chapter.draft"]` |
| `alwaysOnPolicySkills` | `[]` |
| `defaultCapabilityBundles` | `[]` |
| `stateWritePolicy` | `"chapter_confirmed"` |
| `continuityPackDepth` | `"standard"` |
| `chapterGenerationMode` | `"plan_scene_draft"` |
| `windowPlanningHorizon` | `10` |

# Task 3: 用显式持久化契约替代裸 `autoPersist`

> **冲突声明**: 如有冲突，以 `docs/superpowers/specs/2026-04-30-ai-production-system-design.md` 定义为准。

---

## 1. 任务目标

将所有 AI 任务入口中的裸 `autoPersist: true` 替换为显式声明的 `persistMode` 和 `automationTier`，建立清晰的三档自动化控制与三类持久化语义。

### 1.1 当前问题

| 现状 | 问题 |
|------|------|
| `autoPersist: true` | 语义模糊，无法区分写入正式库还是审阅层 |
| 无自动化档位 | 无法区分自动执行还是需用户确认 |
| 风险等级隐式 | 无法快速识别高风险操作 |

### 1.2 目标状态

| 目标 | 收益 |
|------|------|
| 显式声明持久化模式 | 入口语义一目了然 |
| 显式声明自动化档位 | 风险等级可评估 |
| 保留兼容桥 | 不破坏现有调用链 |

---

## 2. 类型体系

### 2.1 PersistMode（持久化模式）

定义三种持久化语义：

| 值 | 语义 | 典型场景 |
|---|------|---------|
| `formal` | 晋升为正式资产，写入 Canon Registry | 蓝图生成、资产种子 |
| `derived_review` | 派生审阅层，记录但不写正式库 | 时间线审阅、关系图审阅 |
| `none` | 仅生成，不持久化 | 草稿预览、一致性检查 |

### 2.2 AutomationTier（自动化档位）

定义三档自动化控制：

| 值 | 语义 | 适用条件 |
|---|------|---------|
| `auto` | 自动执行，用户无感知 | 低风险操作 |
| `supervised` | 执行后通知用户审查 | 正式资产写入 |
| `confirm` | 执行前需用户确认 | 高影响操作 |

# Task 4: 收口全书生成与模块晋升边界，并记录来源轨迹

> **冲突声明**: 如有冲突，以 `docs/superpowers/specs/2026-04-30-ai-production-system-design.md` 定义为准。

---

## 1. 任务目标

将全书生成流程拆分为「蓝图生成」与「资产晋升」两个独立阶段，并为每个正式资产的创建记录来源轨迹（provenance），建立从草案到权威的可追溯链路。

### 1.1 当前问题

| 现状 | 问题 |
|------|------|
| 全书生成跨层执行 | 蓝图生成与资产晋升混在同一流程中 |
| 无来源记录 | 无法追溯正式资产来自哪个 AI 输出 |
| 边界模糊 | 蓝图页同时负责生成和晋升 |
| 审阅层无提示 | 用户无法区分「正式数据」和「派生审阅」 |

### 1.2 目标状态

| 目标 | 说明 |
|------|------|
| 阶段分离 | 全书生成 = 蓝图阶段（8步）+ 晋升阶段（独立操作） |
| 来源可追溯 | 每个正式资产记录其生成来源（蓝图草案 / AI 输出 / 用户输入） |
| 边界收口 | 蓝图页只负责蓝图，资产晋升显式触发 |
| 审阅提示 | 派生审阅层页面显示「本页为审阅层」提示 |

---

## 2. 类型体系

### 2.1 阶段类型

| 阶段 | 语义 | persistMode | automationTier |
|------|------|-------------|----------------|
| `blueprint-first` | 蓝图生成阶段（8步） | `formal` | `supervised` |
| `promote-second` | 资产晋升阶段（独立触发） | `formal` | `confirm` |

### 2.2 来源类型（Provenance Source Kind）

| 值 | 语义 | 典型场景 |
|---|------|---------|
| `blueprint_draft` | 蓝图草案生成 | 蓝图各步骤的 AI 输出 |
| `ai_generation` | AI 直接生成 | 角色/世界/剧情种子 |
| `user_input` | 用户手动创建 | 手动录入的角色 |
| `manual_promotion` | 手动晋升 | 用户确认后晋升 |
| `auto_promotion` | 自动晋升 | supervised 档位自动晋升 |

### 2.3 来源记录字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `entity_type` | `string` | 实体类型（`character` / `world_rule` / `plot_node` 等） |
| `entity_id` | `string` | 实体 ID |
| `source_kind` | `string` | 来源类型（见上表） |
| `source_ref` | `string?` | 来源引用（如蓝图 stepKey / 请求 ID） |
| `request_id` | `string?` | AI 请求 ID |
